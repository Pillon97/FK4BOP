# Project-Alpha-FK4BOP

**Student ID:** FK4BOP

## Project Description and Motivation
This project is a web-based implementation of a classic word-guessing game (Hangman), developed as a full-stack F# application. The motivation behind this project was to explore server-side web development with F# using the WebSharper framework, which allows seamless integration between client and server code within a single language. The application demonstrates how functional programming principles can be applied to build interactive web experiences, including state management, user input handling, and dynamic UI rendering. Beyond serving as a fun and engaging game, the project showcases the practical use of F# for building production-ready web applications and deploying them to a cloud environment (Azure App Service).

## Applied Technologies
The application is built using the F# programming language and the WebSharper framework.
- **Language:** F#
- **Framework:** WebSharper (used for full-stack web development).
- **Project Type:** Web Application.
- **Complexity:** Contains 300+ lines of original F# code (excluding boilerplate, comments, and configuration).

## How to Build and Run
To run this project, you will need the .NET SDK and the WebSharper templates installed on your system.
1. Clone this public repository.
2. Open your terminal and navigate to the project's root directory.
3. Execute the following command: `dotnet run` 
4. The application will be accessible via the localhost address indicated in your terminal.

# Project-Omega-FK4BOP

**Student ID:** FK4BOP

## Project Description and Motivation
This project is a Single Page Application (SPA) built entirely in F#, compiled to JavaScript via the Fable compiler and structured around the Elmish architecture (the Model-View-Update pattern inspired by Elm). The motivation behind this project was to investigate modern client-side web development using functional programming, demonstrating how F# can be used as a robust alternative to mainstream JavaScript and TypeScript frameworks for building reactive user interfaces. By leveraging Fable, Elmish, and React under the hood, the project highlights the benefits of strong static typing, immutability, and predictable state management in front-end development. The application also serves as a practical exercise in setting up a modern toolchain (Vite + npm + .NET SDK) and deploying a statically hosted SPA to GitHub Pages.

## Applied Technologies
The application is built using the F# programming language with the Fable compiler and the Elmish architecture.
- **Language:** F#
- **Framework:** Fable + Elmish (compiled to JavaScript, running as a React-based SPA).
- **Build Tool:** Vite.
- **Project Type:** Single Page Web Application.
- **Complexity:** Contains 300+ lines of original F# code (excluding boilerplate, comments, and configuration).

## How to Build and Run
To run this project, you will need the .NET SDK and Node.js installed on your system.
1. Clone this public repository.
2. Open your terminal and navigate to the `Omega` directory.
3. Install the Node dependencies: `npm install`
4. Start the development server: `npm start`
5. The application will be accessible via the localhost address indicated in your terminal.
